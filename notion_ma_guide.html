Notion System Guide

🗂️ shop.com Notion 系統

建立指南 · 架構說明 · 日常使用情境
適合有基礎 Notion 經驗的夥伴自行建立

01 設計原則

欄位（Properties）

只放需要「篩選、排序、快速掃描」的結構化資料。例如狀態、日期、類型、BV數值。

頁面內文（Body）

詳細說明、長文字、圖片、產品見證照、互動紀錄，全部寫在頁面裡，不塞進欄位。

02 系統架構（7 個資料庫）
👥 人脈庫 — 核心樞紐，所有 DB 都 relation 到這裡
├── ✅ All Task — 每日任務中心
├── 🛍️ 零售庫 — BV / IBV 記錄
│ └── 🏪 夥伴商店庫 — IBV 商店管理
└── 📖 Learn — 課程 / 會議記錄
└── 📚 知識庫 — 可複用知識點
└── 🧴 產品庫 — 產品資訊與見證
紫色 = 核心樞紐｜藍色 = 人脈庫的直接關聯 DB｜綠色 = 二層關聯 DB
箭頭方向代表「被 relation 的方向」，例如 All Task → 人脈庫 表示 All Task 裡有欄位指向人脈庫

04 建立步驟
1

👥 建立人脈庫

全新建立

整套系統的核心樞紐，務必最先建立。其他所有 DB 的 relation 都指向這裡，如果先建其他 DB，relation 欄位就無法正確設定目標。

欄位清單
姓名 title 身份 multi-select 溫度 select 認識管道 select 最後聯繫 date 關聯訂單 → 零售庫 關聯任務 → All Task 關聯學習 → Learn
💡 提示：relation 欄位（關聯訂單、關聯任務、關聯學習）可先留著，等後續 DB 建完後再回來設定 relation 目標。
2

✅ 建立 All Task

全新建立

所有待辦事項的唯一入口。不管是跟進顧客、課後功課、招募任務，全部統一建在這裡，不另開其他清單，才能保持系統整潔。

欄位清單
Name title Date date Status status Type select Source select Schedule checkbox 關聯人脈 → 人脈庫
💡 Type 選項：零售 / 學習 / 招募 / 跟進 / 其他
💡 Source 選項：社群挑戰 / 週計劃 / 手動　（新增任務時，Source 一律選「手動」）
⚠️ 建完後：回到人脈庫，把「關聯任務」relation 欄位的目標設定為 All Task。
3

🛍️ 建立零售庫

全新建立

記錄所有零售交易，分兩種類型：顧客購買shop.com 獨家品牌產品（BV 購買）、自己或顧客在夥伴商店消費（IBV 消費）。兩種都要記，才能完整追蹤 MUFO 進度。

欄位清單
品項/訂單名稱 title 類型 select 顧客 → 人脈庫 日期 date 金額 number BV number IBV number 狀態 select 關聯商店 → 夥伴商店庫
💡 類型選項：BV購買 / IBV消費　｜　狀態選項：待確認 / 已下單 / 完成
⚠️ 注意：「關聯商店」先建欄位即可，等第 7 步夥伴商店庫建完後再回來設定 relation 目標。建完後也記得回人脈庫把「關聯訂單」設定好。
4

📖 建立 Learn

全新建立

所有學習事件的原始記錄，包含 Coring 課程、mentor 會議、團隊會議、錄音檔。上完課的重點筆記和待辦功課都寫在頁面內文，提煉後的知識點再移進知識庫。

欄位清單
課程/會議名稱 title 類型 select 日期 date 附件/連結 URL 講師/對象 → 人脈庫 產出知識條目 → 知識庫
💡 類型選項：課程 / mentor會議 / 團隊會議 / 錄音檔
⚠️ 注意：「產出知識條目」先建欄位即可，等第 5 步知識庫建完後再設定 relation 目標。建完後回人脈庫把「關聯學習」設定好。
5

📚 建立知識庫

全新建立

存放從 Learn 提煉出的可複用知識點，一筆 Learn 課程可產出多筆知識條目。分類管理後，未來跟顧客或新夥伴分享時可以快速查找。

欄位清單
條目名稱 title 分類 select 來源課程 → Learn 來源講師 → 人脈庫 標籤 multi-select
💡 分類選項：態度與知識 / 目標設定 / 零售 / 物色篩選 / 跟進ABC　｜　標籤：話術 / 心態 / 行動
⚠️ 建完後：回到 Learn，把「產出知識條目」relation 欄位的目標設定為知識庫。
6

🧴 建立產品庫

全新建立

shop.com 獨家品牌 BV 產品的知識管理中心。欄位負責結構化資訊（系列、BV值、適用對象），頁面內文則存放核心賣點、個人使用見證（含照片）、夥伴分享的他人見證。

欄位清單
產品名稱 title 系列 select 分類 select BV值 number 適用對象 multi-select 關聯知識條目 → 知識庫 關聯零售記錄 → 零售庫
💡 系列選項：Isotonix / TLS / Motives / 其他　｜　分類：健康 / 美容 / 家居 / 其他
💡 建立方式：不需要一次建完所有產品，從自己最熟悉或最常推薦的 3–5 個產品開始，逐步補充。
7

🏪 建立夥伴商店庫

最後完成

記錄所有 SHOP.COM 合作夥伴商店，方便推薦顧客到對應商店消費以產生 IBV。用「分類」或「適合推薦給」篩選，可以快速找到最適合推薦的商店。

欄位清單
商店名稱 title 分類 select IBV說明 text 適合推薦給 multi-select 關聯零售記錄 → 零售庫
💡 分類選項：餐飲 / 電商 / 旅遊 / 日用品 / 娛樂 / 其他　｜　適合推薦給：外食族 / 旅遊族 / 網購族 / 全部
⚠️ 最後收尾：建完後回到零售庫，把「關聯商店」relation 欄位的目標設定為夥伴商店庫，整套系統正式完成。

→ 夥伴商店庫頁面記體驗心得；零售庫建 IBV 消費記錄，relation 到該商店
→

→ All Task 開 Filter，Date = Today，掃描未完成任務
→
